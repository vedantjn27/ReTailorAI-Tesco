## 🌐 What is ReTailor AI?

**ReTailor AI** is a full-stack, AI-powered creative production platform purpose-built for the **$140B+ global retail media advertising market**. It compresses a workflow that once took weeks and thousands of dollars in agency fees into a guided, intelligent pipeline that any brand — designer or not — can complete in under five minutes.

It is not a template picker. It is not a basic image editor. It is a **complete generative AI creative engine** — from raw product photo to fully validated, multi-format, campaign-ready assets — powered by **Mistral Large LLM**, **HuggingFace FLUX image generation**, **rembg neural background removal**, **OpenCV smart enhancement**, and a **MongoDB GridFS** binary asset store, all served through a high-performance **FastAPI** backend and a **Next.js 16** frontend with real-time collaboration.

## 🚀 Features

### 🖼️ Feature 1 — AI Creative Builder & Asset Manager

The foundation of every creative workflow. Upload product images, logos, and brand assets into a **MongoDB GridFS** store for scalable binary management.

**What's inside:**
- **Neural background removal** via `rembg` — industry-leading AI segmentation with automatic **OpenCV GrabCut** fallback for edge cases
- **6-Stage Smart Enhancement Pipeline:**
  1. Auto white balance correction
  2. Shadow and highlight recovery (HDR-style processing)
  3. Intelligent auto contrast (CLAHE-based)
  4. Noise reduction (bilateral filtering)
  5. Smart adaptive sharpening
  6. Optional HD upscale for print-ready output
- **Smart Crop** — subject-aware, focal-preserving crop with mode presets: `tight` · `square` · `portrait` · `landscape` · `amazon` · `custom`
- **Full CRUD asset management** with GridFS-backed streaming delivery
- Temporary local processing with `tmp/` and `processed/` directories for pipeline efficiency

---

### 🎨 Feature 2 — Drag-and-Drop Canvas Editor

A fully layered, persistent canvas editing system backed by MongoDB project storage.

**Capabilities:**
- **Image layers and text layers** with full transform control: position (x/y), rotation, opacity, scale, width, height
- **Server-side canvas rendering** — PIL/Pillow composes all layers into a single PNG, ensuring pixel-perfect consistency regardless of the client device
- **Full undo/redo** via server-side state management (history and future stacks per project)
- **Template application** — overlay retailer-compliant templates onto any active project; layers merge intelligently
- **Multi-device preview** — desktop, mobile, in-store display simulations
- **Grid snapping and layer ordering** for precise, professional composition
- Project persistence in MongoDB — resume any creative from any device

---

### 🤖 Feature 3 — AI Layout & Design Suggestions

Powered by **Mistral Large**, the AI suggestions engine analyzes your uploaded product image, logo, headline, and CTA text to generate complete, platform-optimized layout recommendations — not generic suggestions, but layouts calibrated to your specific assets.

**How it works:**
- Submit product image ID, logo ID, headline text, and CTA copy
- Mistral analyzes brand context and generates multiple complete layout variants
- Each variant includes: element positions, color harmony recommendations, typography guidance, and channel-specific notes
- **One-click application** of any layout to the current canvas project via `/editor/{project_id}/apply-template/{template_id}`
- Results scored and ranked by estimated compliance rating

---

### 🤖 Feature 4 — AI Poster Generator (Document → Creative)

A standalone, document-to-finished-creative pipeline. The most innovative feature in the platform.

**The pipeline:**
1. User uploads a **PDF or DOCX** file (promo brief, product spec, campaign document) or pastes a text prompt
2. **PyPDF2 / python-docx** extracts all text content
3. **Mistral Large** reads the document and engineers a detailed, tailored image generation prompt — explicitly specifying visual layout, typography style, text placement, color aesthetic, and banner composition
4. The enriched prompt is sent to the **HuggingFace FLUX model** via Inference API
5. A production-quality poster image is rendered and stored in **MongoDB GridFS**
6. The result is returned as a canvas-ready asset that can immediately enter the editor workflow

**What makes it unique:** Mistral doesn't just summarize the document — it specifically crafts a FLUX prompt that quotes the exact copy to be rendered on the poster, instructs the model on layout, and aligns aesthetics with the brand context extracted from the document.

---

### ✅ Feature 5 — Real-Time Compliance Validator

A rule engine that checks every layer of every project against configurable, retailer-specific compliance rule sets stored in MongoDB.

**Checks performed per layer:**
- Minimum font size enforcement (retailer-specific threshold)
- Text contrast ratio — **WCAG AA standard** (4.5:1 minimum) computed via `contrast_ratio()` against sampled background color
- Logo safe-margin validation — minimum margin computed as percentage of canvas short side
- Background lightness requirements
- Color scheme restrictions per retailer
- Layer position and inter-element spacing rules

**Severity tiering:** `low` · `medium` · `high` — with per-violation layer attribution for precise issue location

---

### 🔧 Feature 6 — Compliance AutoFix

One click. All fixable violations corrected automatically.

**What AutoFix handles:**
- Font size bumped to retailer minimum threshold
- Text color switched to highest-contrast option (black/white) by sampling actual background pixel color behind each text layer
- Logo repositioned to respect safe margins (computed dynamically per canvas size)
- All changes logged with **before/after diff** for full audit trail
- Non-destructive — original project preserved in undo history

---

### ✍️ Feature 7 — Mistral AI Compliance Advisor

Beyond rule-based error codes, a natural-language compliance guide powered by **Mistral Large**.

- Takes a list of compliance violations as structured input
- Returns **human-readable, actionable guidance** tailored to each violation type — what it means, why it matters, and how to fix it creatively
- Designed for non-designer users who need context, not just error codes
- Accessible via `/ai/compliance-advice` — fully integrated into the compliance dashboard

---

### ✍️ Feature 8 — AI Copy & Headline Generator

**Mistral Large** generates retail-optimized ad copy, headlines, sub-copy, and CTA text on demand.

**Inputs:** product name · key benefit · target audience · desired tone  
**Outputs:** multiple headline variants · sub-copy options · CTA alternatives  
**Calibrated for:** retail media platforms — concise, benefit-led, guideline-safe messaging  
**API endpoint:** `/ai/copy-suggestions`

---

### 📤 Feature 9 — Multi-Format Export & Batch Pipeline

One-click generation of every required format variant from a single master creative.

**What's included:**
- **Auto-Resize** generates 30+ platform formats: Facebook (feed, story, carousel) · Instagram · Google Display Network · Amazon DSP · in-store digital screens · print banners · and more
- All resizes are **focal-preserving** — the product subject is never accidentally cropped
- **Batch Export** packages all variants into a single downloadable ZIP from GridFS
- **File optimization** — lossless/lossy JPEG/PNG compression with configurable quality targets
- Print-ready output with configurable bleed and trim marks
- All assets stored in GridFS with streaming download endpoint

---

### 🤝 Feature 10 — Collaboration & Review Workspace

A built-in async review system that eliminates the need for external annotation tools.

**Full capability set:**
- **Inline annotations** with precise bounding-box coordinates mapped to canvas position (not pixel-space coordinates — canvas-space, so they survive resize)
- **Version-linked comments** — every annotation is tied to a specific version number
- **Comment CRUD** — add, view, update, delete reviews with full user attribution
- **Real-time updates** via **WebSocket** connections — reviewers see annotations appear live
- **Shareable review links** for external stakeholders who don't have platform accounts

---

### 📁 Feature 11 — Template Library

A seeded, extensible library of production-validated retailer-compliant templates.

- Pre-built templates for major retail media platforms (Tesco, Amazon, Meta, Google)
- Each template stores a full layer structure, canvas dimensions, background color, and compliance metadata
- **One-click apply** to any existing project — layers are merged intelligently
- **Admin seeding** via `/templates/reseed` and `/templates/add` — extensible without code changes
- Templates queryable by ID or full list — browsable in the frontend template gallery

---

### ⚙️ Feature 12 — Campaign Dashboard & Workspace

A unified project management and analytics layer across all creative work.

- **Dashboard** with campaign status indicators, recent projects, creative output metrics, and quick action shortcuts
- **Campaign workspace** — group creatives by campaign, track version history, monitor approval status
- **Settings page** — theme management (dark/light), API key configuration, user preferences
- **Demo Mode** — the full platform experience using a mock API client (`demo-api-client.ts`) with zero backend dependency — ideal for presentations and onboarding
- **Vercel Analytics** integration for usage tracking

## 🔄 System Workflow

```
PHASE 1 ─ ASSET INGESTION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
User uploads product image or logo
  → POST /upload-asset → MongoDB GridFS storage
  → rembg neural background removal
  → (fallback) OpenCV GrabCut
  → 6-stage smart enhancement pipeline
  → Smart crop with focal-point preservation

PHASE 2 ─ CANVAS COMPOSITION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
POST /editor/create-project → new project in MongoDB
  → POST /editor/{id}/add-image-layer
  → POST /editor/{id}/add-text-layer
  → PUT /editor/{id}/update-layer/{layer_id}
  → Apply retailer template via /editor/{id}/apply-template/{template_id}
  → Server renders PNG: GET /editor/{id}/render → Pillow composition
  → Undo: POST /editor/{id}/undo | Redo: POST /editor/{id}/redo

PHASE 3 ─ AI INTELLIGENCE LAYER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
POST /ai/layout-suggestions
  → Mistral Large: analyzes assets, generates layout variants
  → Returns multiple ranked layout options

POST /ai/copy-suggestions
  → Mistral Large: generates headlines, sub-copy, CTA options

POST /ai/compliance-advice
  → Mistral Large: plain-English violation explanations

POST /generate-poster
  → PyPDF2/python-docx: extract document text
  → Mistral Large: engineer detailed FLUX image prompt
  → HuggingFace FLUX API: render poster image
  → Store in GridFS → return canvas-ready asset

PHASE 4 ─ COMPLIANCE VALIDATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
GET /compliance/check/{project_id}?retailer=RetailerA
  → Load retailer rule set from MongoDB
  → Per-layer checks: font size, contrast ratio, logo margin, colors
  → Return severity-tiered violations (low/medium/high)

POST /compliance/autofix/{project_id}
  → Apply automated fixes to all layers
  → Log before/after diff for audit trail
  → Non-destructive (original preserved in history)

PHASE 5 ─ EXPORT PIPELINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
GET /auto-resize/{file_id}
  → Generate 30+ platform format variants
  → Focal-preserving crop logic per format

POST /batch-export
  → Package all variants into ZIP
  → File optimization (JPEG/PNG compression)
  → GridFS storage → streaming download

PHASE 6 ─ REVIEW & COLLABORATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
POST /review/add → annotation with canvas bounding box
  → WebSocket broadcast to all connected reviewers
GET /review/{file_id} → all annotations for asset
PUT /review/update/{review_id} → edit annotation
DELETE /review/delete/{review_id} → remove
  → Team approval → campaign delivery
```

## ⚡ Quick Start

### Prerequisites

| Tool | Min Version |
|---|---|
| Python | 3.12+ |
| Node.js | 18.0+ |
| MongoDB Atlas account | Free tier |
| Mistral AI API key | Free at console.mistral.ai |
| HuggingFace API key | Free at huggingface.co |

### 1. Clone
```bash
git clone https://github.com/your-org/retailor-ai.git
cd retailor-ai
```

### 2. Backend setup
```bash
cd backend
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp ../.env.example .env           # Fill in your API keys
uvicorn main:app --reload --port 8000
```

### 3. Frontend setup
```bash
cd frontend
npm install
npm run dev
```

### 4. Open the platform
- **App:** http://localhost:3000
- **API docs (Swagger):** http://localhost:8000/docs
- **Health check:** http://localhost:8000/

> 📘 Full setup including Docker, cloud deployment, and API key guides: **[`SETUP_GUIDE.md`](SETUP_GUIDE.md)**
